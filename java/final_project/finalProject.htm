Java Class Project

 

This project is divided up into multiple sections, please read through all sections. The topic of the program is your choice (see the end of this document for some ideas), however, it must encompass techniques we have covered throughout the quarter. The minimum features your program must include are: 

v    User interaction - via a GUI

v    All logic structures - sequence, selection and repetition

v    A user defined method

v    An event handler

v    Data validation

v    Use objects and members of a class

v    Internal documentation (including comments)

v    Database or Sequential file interaction

 

 You will receive bonus points for including:

v    Array(s)

v    Arithmetic

v    Counters/accumulators

v    Built-in math method or random number generator

 

 The program must contain internal documentation including appropriate header information, meaningful variable declarations, and informative commenting.
 

 

Development Phase
Due Week 7:

Define and limit the overall purpose of your program. What problem are you solving? Analyze the scope of the problem to be solved (ie. what will and will NOT be included in your program). Design the entire program, include either flowcharts or pseudo code along with UML diagrams (see page 340 of your textbook - you will want to review UML diagrams before we actually cover them in class - they are introduced in Chapter 6 and also covered in Chapter 8). If you will be producing a report (or any other type of output)- show what the report (or output) will look like. Define the types of code that will be utilized in each part of the program. Turn this into the instructor for feedback. Prepare to make modifications depending on your instructor’s feedback.

If you would like feedback on the feasibility of your project before Week 7, send your Development Phase to me via email.

Please note: Even though you will have until Week 9 to complete your program, do not wait. It will take a lot of time to build (and get to work). The more time you put into up front planning and charting, the easier and faster your development work will go. A good rule is to put 80% of your work into the logic/charting of your program and 20% of your time coding. Start working on it now and work on it steadily throughout the next few weeks.

Additional notes: MS Word has built-in flowchart shapes. MS Visio (which is a free download from MSDNAA) is an easy to use/learn charting package (recommended over MS Word). See the Project Sheet Explained document for more information about this portion of the project.

 

Code and Test Phase
Due Week 9:

Code, document, test, and if necessary, modify the program. Is your application user friendly (as opposed to user hostile)? Does it have helpful and meaningful error messages, zero blow ups, lock outs or non-functions? Is your code understandably commented and documented? This means that your code must be completed and ready to go by now.

Create a list of your program's features to check against while testing. The checklist will be used by you and your Group members. You might want to use the Grading of Final Project worksheet as your checklist, modified for your project. Your code should be thoroughly tested by you before you have Group members test it.

A signed testing sheet by at least 2 classmates saying that they have tested your code and it works as promised constitutes a sign-off. Be sure to leave comments for code that you are testing about parts of the program that don't work as promised or ideas for improvements. You will have until Week 10 to make changes.

 

Presentation Phase 
Due Week 10:

 Submit your Final Presentation in the Assignment Area, it will include: 

v    The purpose of your code (what problem does it solve).  

v    Your modified and updated design work (logic and UML) . 

v    Your code (all of the solution files). 

v    Lessons Learned - what did you learn from this project, what would you do different next time? 

v    How would you improve the next release of your program?

 

Some Project Ideas: 

Simple Business Application: You may want to borrow ideas from other classes, texts or your own office experience: Using just the concepts in our text, you could make quite a few business simple applications such as “payroll”, “accounts receivable/payable”, “budget”, “time-clock”, “bank reconciliation” program, “income statement”. 

Financial: You might create a program that helps a “real-estate rep” or a “car salesperson” keep track of clients and sales. 

Games: You may want to think of “games” to create. Games that make the user solve problems (simple math or questions) and, depending on their input, suffer (or enjoy) different “consequences”. A simple game of hangman with a "graphic" representation of the "consequences" for each incorrect answer.
